Jika Anda mengklik tombol "Kirim", data formulir akan dikirim ke halaman bernama "/action_page.php".

Kesimpulan Ujicoba Form:

1. Tombol Reset:
Ketika tombol "Reset" ditekan, semua isian pada form akan kembali ke nilai awalnya (value default). Jika tidak ada value default, maka input akan dikosongkan. Reset tidak mengirim data ke server, hanya mengatur ulang tampilan form.

2. Tombol Kirim:
Ketika tombol "Kirim" ditekan, data dari semua input dalam form akan dikirim ke halaman yang ditentukan pada atribut action ("/action_page.php"). Jika file action_page.php tidak ada, browser akan menampilkan halaman error 404.

3. Atribut target="_self":
Hasil form akan dibuka di tab/halaman yang sama (default). Dengan target="_blank", hasil akan dibuka di tab baru. Dengan target="_parent", hasil dibuka di frame induk. Dengan target="_top", hasil dibuka di seluruh jendela browser.

4. Atribut method="get" dan method="post":
Dengan method="get", data form akan terlihat di URL browser dalam bentuk query string (contoh: /action_page.php?i_name=Budi&i_belakan=Prawira). Data GET tersimpan di riwayat browser dan dapat di-bookmark, namun tidak aman untuk data sensitif dan memiliki batas panjang karakter.
Dengan method="post", data form dikirim tersembunyi di dalam badan HTTP request sehingga tidak terlihat di URL. POST lebih aman untuk data sensitif seperti password, tidak tersimpan di riwayat browser, tidak dapat di-bookmark, dan tidak memiliki batas panjang data.
